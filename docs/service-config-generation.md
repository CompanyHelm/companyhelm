# Service config generation

This is the simple version of how CompanyHelm generates local config when you run `companyhelm up`.

## Where files live

CLI-generated files now live under:

- `~/.companyhelm/cli/`

That includes:

- `~/.companyhelm/cli/config.yaml`
- `~/.companyhelm/cli/github-app.yaml`
- `~/.companyhelm/cli/runtime/api-config.yaml`
- `~/.companyhelm/cli/runtime/frontend-config.yaml`
- `~/.companyhelm/cli/runtime/api/.env`
- `~/.companyhelm/cli/runtime/docker-compose.yaml`
- `~/.companyhelm/cli/runtime/state.yaml`

If `COMPANYHELM_HOME` is set, CompanyHelm uses that path instead for runtime files.

Repo-owned image pins stay in the repo root in:

- `config.yaml`

## What generates them

`companyhelm up` writes the runtime files directly from the CLI.

There are two main inputs:

- runtime state generated by the CLI, such as ports, auth keys, runner secret, and bootstrap data
- optional GitHub App credentials saved by `companyhelm setup-github-app`

The machine GitHub App config is stored in:

- `~/.companyhelm/cli/github-app.yaml`

If `COMPANYHELM_CONFIG_HOME` is set, that path is used instead.

## API

The API uses:

- `~/.companyhelm/cli/runtime/api-config.yaml`
- `~/.companyhelm/cli/runtime/api/.env`

`api-config.yaml` is generated during `companyhelm up`.
It includes the app port, database settings, JWT settings, and GitHub settings.

When GitHub App config exists, the API YAML keeps these values as env references:

- `${GITHUB_APP_CLIENT_ID}`
- `${GITHUB_APP_PRIVATE_KEY_PEM}`
- `${GITHUB_APP_URL}`

Those env vars are written into `~/.companyhelm/cli/runtime/api/.env` from `src/templates/api.env.tpl`.

If GitHub App config does not exist, the `.env` file is still generated with blank values and the API YAML uses disabled placeholder values instead.

## Web

The web service uses:

- `~/.companyhelm/cli/runtime/frontend-config.yaml`

This file is also generated during `companyhelm up`.
It includes the GraphQL API URL, runner gRPC target, and auth provider settings.

There is no separate generated `.env` file for `web` in this repo.

## How they are used

In Docker mode:

- the generated `docker-compose.yaml` mounts `api-config.yaml` into the API container
- it loads `~/.companyhelm/cli/runtime/api/.env` with `env_file` for the API container
- it mounts `frontend-config.yaml` into the frontend container

In local repo mode:

- the local API process starts with `--config-path ~/.companyhelm/cli/runtime/api-config.yaml`
- the local API process also receives the GitHub App env vars directly from the saved machine config
- the local web repo runs `npm run config:generate -- --config-path ~/.companyhelm/cli/runtime/frontend-config.yaml` before starting

## Repo image pins

Image pinning is repo-owned, not CLI-owned.

`companyhelm set-image-version` writes image pins to:

- `companyhelm.yaml`

Those values are read when CompanyHelm resolves which API and frontend images to use.

## Reset behavior

`companyhelm reset` removes the generated runtime files under `~/.companyhelm/cli/runtime` and the CLI workspace config at `~/.companyhelm/cli/config.yaml`.
It does not remove the repo-owned `companyhelm.yaml` file.
