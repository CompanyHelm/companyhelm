# Service config generation

This is the simple version of how CompanyHelm generates local service config when you run `companyhelm up`.

## What gets generated

`companyhelm up` writes three files used by `api` and `web`:

- `~/.companyhelm/api-config.yaml`
- `~/.companyhelm/frontend-config.yaml`
- `./.companyhelm/api/.env`

If `COMPANYHELM_HOME` is set, CompanyHelm uses that instead of `~/.companyhelm` for the runtime files.

## Where the values come from

There are two main inputs:

- Runtime state generated by the CLI, such as ports, auth keys, runner secret, and bootstrap data
- Optional GitHub App credentials saved by `companyhelm setup-github-app`

The GitHub App config is stored machine-wide in:

- `~/.config/companyhelm/github-app.yaml`

If `COMPANYHELM_CONFIG_HOME` is set, that path is used instead.

## API config

The API gets:

- `~/.companyhelm/api-config.yaml`
- `./.companyhelm/api/.env`

`api-config.yaml` is generated directly by the CLI during `companyhelm up`.
It includes the app port, database connection settings, JWT settings, and GitHub settings.

When GitHub App config exists, the YAML keeps GitHub values as env references:

- `${GITHUB_APP_CLIENT_ID}`
- `${GITHUB_APP_PRIVATE_KEY_PEM}`
- `${GITHUB_APP_URL}`

Those env vars are written into `./.companyhelm/api/.env` from `src/templates/api.env.tpl`.

If no GitHub App config exists, the `.env` file is still generated, but the values are empty.
The API YAML falls back to disabled placeholder values instead of using the env references.

## Web config

The web service gets:

- `~/.companyhelm/frontend-config.yaml`

This file is generated directly by the CLI during `companyhelm up`.
It includes the API GraphQL URL, runner gRPC target, and auth provider settings.

There is no separate generated `.env` file for `web` in this repo.

## How the files are used

In Docker mode:

- the generated `docker-compose.yaml` mounts `api-config.yaml` into the API container
- it loads `./.companyhelm/api/.env` with `env_file` for the API container
- it mounts `frontend-config.yaml` into the frontend container

In local repo mode:

- the local API process starts with `--config-path ~/.companyhelm/api-config.yaml`
- the local API process also receives the GitHub App env vars directly from the saved machine config
- the local web repo runs `npm run config:generate -- --config-path ~/.companyhelm/frontend-config.yaml` before starting

## Regeneration

These files are generated again each time you run `companyhelm up`.

`companyhelm reset` also resets `./.companyhelm/api/.env` back to the template version.
